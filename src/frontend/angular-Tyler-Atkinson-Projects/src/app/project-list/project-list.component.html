Traveling Salesman

A Project for CSC 340: Scientific Computing. Used two different methods for solving the classic Traveling Salesman Problem in Java.

Genetic Algorithm: Simulates evolution, representing each city as a gene. Fitness is measured by 1/distance, where routes with smaller distances are more likely to pass on their genes. You can learn more about Genetic Algorithms here.

Simulated Annealing: Simulates annealing used in metallurgy. It begins with a high temperature and as it runs the temperature is cooled. It continues to run until the temperature has completely cooled. When the temperature is high, the algorithm is more likely to accept a new route. When the temperature is low, the opposite is true. The slower the cooling rate, the more likely that the algorithm will find the best route. You can learn more about Simulated Annealing here.

You can experiment with the two implementations here.
