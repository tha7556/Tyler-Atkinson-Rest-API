Maze Solver

Background

Mazes are very simple puzzles that almost everyone is familiar with. This makes it very interesting to watch run, especially for those not very familiar with the programming concepts behind it. Here my partner and I implemented 3 different algorithms to solve mazes, each of which we called Maze Runners.

The Mazes

We tried to focus more on the solving of mazes, rather than their generation. So we used the algorithm from here to create our mazes. Each maze is simply connected, meaning that there are no loops and that all walls are connected. We had 4 different categories for mazes:

  • Small - 51x51 verticies
  • Medium - 101x101 verticies
  • Large - 201x201 verticies
  • Crazy - 1467x1467 verticies

For each of these categories, we generated 200 samples to ensure that we had plenty of test data.

Junction Origination Runner

Our first runner is based on junction origination. Whenever it is faced with a choice it picks one at random. It continues to do so until it encounters a dead-end. Once it does, it backtracks to the most recent junction.

In the above GIF, whenever it has to backtrack the junction is marked with a light blue square. Once it finds the correct path it begins highlighting it in green.

Wall Follower Runner

The Wall Follower Runner is set to follow either the left or the right wall. Because the mazes are simply connected, it is able to follow the wall all the way until the end of the maze.

Manhattan Distance Runner

This runner was given a bit more information than the others, probably resulting in an unfair advantage. This runner knows the coordinates of the end of the maze, just not the path to it. Whenever it encounters a junction, it calculates how far each choice is from the goal and picks the one with shortest distance. Similarly to the Junction Origination, if it hits a dead end it backtracks to the last junction.

Results

Results were measured by comparing the number of steps taken by each of the runners on the same maze. As you can see above, the Manhattan Distance runner performs the best almost everytime, as expected. The Junction Origination is pretty random. In some cases it gets the correct path immediately, other cases the correct path is the last one that it tries. The left and right Wall Followers are always the inverse of each other, because they make opposite decisions from each other in every situation.
