Sorting Visualizations

Background

Sorting data is a common problem in computing. As such, there are many different algorithms to solve this problem.

Quick Sort

Quick Sort, like the name suggests, is a very efficient way to sort data. It is a recursive algorithm that subdivides the array into smaller arrays before sorting. Due to its speed, it is used as the default sorting function in many languages. To learn more about Quick Sort click here.

Selection Sort

Selection Sort is a simple algortihm that specializes in sorting with limited memory. It goes through each element in the array, and swaps it with the smallest element that hasn't already been sorted. It is very easy to implement, but does not handle large datasets well. To learn more about Selection Sort click here.

Heap Sort

Heap Sort is similar to Selection Sort, except instead of iterating through each entry to find the next smallest element, it uses a heap to locate it. To learn more about Heap Sort click here.

Bubble Sort

Bubble Sort is renowned as one of the worst sorting algorithms. It continues to pass through the array and checks two values at a time. If those two elements are out of order then they are swapped. This continues until no element is out of place. To learn more about Bubble Sort click here.
